# Bonafide Certificate — Design Spec

**Date:** 2026-06-04  
**Status:** Approved  

---

## Overview

Add bonafide certificate generation to the ERP. Admins and principals can generate a print-ready certificate for any student from a dedicated Certificates page. Certificates follow the school's letterhead template (logo, name, address), are downloaded via browser print-to-PDF, and every issuance is logged in the database for audit purposes.

---

## Scope

### In scope
- Schema: add `date_of_birth`, `parent_name`, `gender` to `student_profiles`
- Schema: new `bonafide_certificates` audit log table
- Settings page: add logo upload + address fields
- Student edit form: add DOB, parent name, gender fields
- Certificates list page: `/admin/certificates` and `/principal/certificates`
- Certificate detail/print page: `/admin/certificates/[studentId]` and `/principal/certificates/[studentId]`
- Audit log saved on every generation

### Out of scope
- Parent-initiated certificate requests (future)
- Push/email delivery to parents (future)
- Transfer certificates or other document types (future)
- Separate school seal image (reuse logo_url as watermark)

---

## Schema Changes

### 1. `student_profiles` — new columns

```sql
ALTER TABLE public.student_profiles
  ADD COLUMN date_of_birth DATE,
  ADD COLUMN parent_name   TEXT,
  ADD COLUMN gender        TEXT CHECK (gender IN ('male', 'female', 'other'));
```

All nullable — existing students are unaffected.

### 2. `bonafide_certificates` — new table

```sql
CREATE TABLE public.bonafide_certificates (
  id                 UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  school_id          UUID NOT NULL REFERENCES public.schools(id) ON DELETE CASCADE,
  student_profile_id UUID NOT NULL REFERENCES public.student_profiles(id) ON DELETE CASCADE,
  academic_year_id   UUID NOT NULL REFERENCES public.academic_years(id),
  generated_by       UUID NOT NULL REFERENCES auth.users(id),
  generated_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

RLS: school_admin and principal can insert/select for their school. No update/delete.

## Settings Page Additions (`/admin/settings`)

Two new fields added to the existing form:

- **School Logo** — image upload (PNG/JPG, max 2MB). Uploads to Supabase Storage bucket `school-assets` at path `{school_id}/logo`. Saves public URL to `schools.logo_url`.
- **School Address** — multiline textarea. Saves to `schools.address`. Supports multiple address lines (e.g., KG Campus and High School Campus).

---

## Student Edit Form Additions

Three new optional fields added to the student edit form (used in admin student profile edit):

| Field | Type | Notes |
|---|---|---|
| Date of Birth | Date picker | Stored as `DATE` in DB |
| Parent Name | Text input | e.g., "RAMESH A" — used as-is in certificate |
| Gender | Dropdown: Male / Female / Other | Drives son/daughter/child and His/Her/Their in certificate |

## Certificates Page (`/admin/certificates`, `/principal/certificates`)

Mirrors the report cards page pattern.

**Two tabs:**

1. **Students** — searchable table of all active enrolled students with columns: Name, Class, Section, Admission No. Each row has a "Generate" button.
2. **History** — table of all `bonafide_certificates` records for this school, showing: Student Name, Class, Academic Year, Generated By, Date Issued.

**Access control:**
- `school_admin` role: full access at `/admin/certificates`
- `principal` role: full access at `/principal/certificates`
- Teachers: no access

---

## Certificate Print Page (`/admin/certificates/[studentId]`, `/principal/certificates/[studentId]`)

Server component that fetches all required data, renders a `CertificateView` client component.

**Data fetched:**
- `schools`: `name`, `logo_url`, `address`
- `student_profiles`: `full_name`, `admission_number`, `date_of_birth`, `parent_name`, `gender`
- `student_enrollments`: `roll_number`, `class`, `section` (active year)
- `academic_years`: `name` (active year)

**On render:**
- Displays the certificate with a "Print / Download PDF" button
- The audit log record (`POST /api/certificates/log`) is saved when the user clicks "Print / Download PDF" — not on page load — to avoid duplicate records from page refreshes

### Certificate layout

```
┌─────────────────────────────────────────────────────┐
│  [LOGO]   SCHOOL NAME (large, styled)               │
│           Address line 1                            │
│           Address line 2                            │
│  ─────────────────────────────────────────────────  │
│                                                     │
│           TO WHOM IT MAY CONCERN                    │
│                                                     │
│    This is to certify that [NAME], a student of     │
│    [Class X – A], Adm No. [1316] [son/daughter/     │
│    child] of Mr./Mrs./[PARENT NAME] is a bonafide   │
│    student of our school for the academic year      │
│    [2025-26]. His/Her/Their date of birth is        │
│    [DD/MM/YYYY] as per our school records.          │
│                                                     │
│              [LOGO watermark, low opacity]          │
│                                                     │
│  Date: DD/MM/YYYY                    PRINCIPAL      │
└─────────────────────────────────────────────────────┘
```

**Gender-derived text:**

| Gender | Pronoun | Salutation |
|---|---|---|
| male | son / His | Mr. |
| female | daughter / Her | Mrs. |
| other / null | child / Their | Mr./Mrs. |

**Print implementation:** Same pattern as report cards — `window.open` new tab with injected HTML + CSS, `window.print()`. No external PDF library needed.

## API Route

**`POST /api/certificates/log`**

- Auth: requires valid session with `school_admin` or `principal` role
- Body: `{ student_profile_id, academic_year_id }`
- Action: inserts a `bonafide_certificates` row
- Returns: `{ id }` of created record

## File Structure

```
apps/web/
  app/
    (school)/
      admin/
        certificates/
          page.tsx                  ← list + history tabs
          [studentId]/
            page.tsx                ← data fetching server component
            certificate-view.tsx    ← print-ready client component
      principal/
        certificates/
          page.tsx                  ← same as admin version
          [studentId]/
            page.tsx
            certificate-view.tsx    ← shared from admin (re-export or shared component)
    api/
      certificates/
        log/
          route.ts                  ← POST audit log entry
  supabase/
    migrations/
      20240001000032_bonafide_certificates.sql
```

The `certificate-view.tsx` component is defined once under `admin/certificates/[studentId]/` and imported by the principal route to avoid duplication.

## Missing Data Handling

If a student is missing `date_of_birth`, `parent_name`, or `gender`:
- Certificate still generates — missing fields show as `"—"` or fall back to neutral phrasing
- A yellow warning banner is shown above the certificate: "Some student details are missing. Edit the student profile to fill them in before printing."

---

## Out of Scope (Explicitly Deferred)

- Parent-side certificate request flow in mobile app
- Push notification / WhatsApp delivery of the PDF
- Digital signature or QR verification code on the certificate
- Transfer certificates or character certificates
